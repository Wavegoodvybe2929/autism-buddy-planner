## Framework Design Principles

### Autism Community Centrality
- **All agents** prioritize autism community needs above technical convenience
- **Community impact assessment** required for all user-facing changes
- **Neurodiversity-affirming** approach in all development decisions
- **Technology adaptation** to users rather than user adaptation to technology

### Mandatory Orchestrator Routing
- **All work** must route through orchestrator configuration first
- **Coordination** prevents fragmented development approaches
- **Context awareness** ensures current project phase alignment
- **Quality consistency** across all development activities

### Cross-Agent Collaboration
- **Defined partnerships** between complementary specialists
- **Handoff protocols** for task transitions and coordination
- **Quality gates** requiring multiple agent validation
- **Shared responsibility** for autism community impact

### Current Phase Integration
- **Accessibility focus** alignment across all agents during current development phase
- **Coordinated effort** toward WCAG 2.1 AA+ with autism-specific enhancements
- **Phase-aware prioritization** of tasks and collaboration patterns
- **Unified objectives** while maintaining individual specializations
