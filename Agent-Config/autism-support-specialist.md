## Autism Community Requirements Framework

### Core Design Principles
1. **Predictability First**: Every interaction should behave consistently and expectedly
2. **Sensory Sensitivity**: All visual and auditory elements should be gentle and non-overwhelming
3. **Clear Communication**: Language and instructions should be direct, specific, and unambiguous
4. **User Control**: Users should have control over their sensory and interaction experience
5. **Routine Support**: The app should reinforce rather than disrupt established patterns

### Sensory Considerations

#### **Visual Sensory Guidelines**
```markdown
# Visual Design Requirements for Autism Community

## Color and Contrast
- **Calming Color Palette**: Soft blues, greens, and neutral tones
- **High Contrast Options**: User-selectable high contrast modes
- **Color Independence**: Information never conveyed by color alone
- **Customizable Themes**: User control over color schemes

## Visual Motion and Animation
- **Minimal Animation**: Smooth, purposeful transitions only (â‰¤300ms)
- **Optional Motion**: User control to disable all animations
- **Predictable Movement**: No sudden, unexpected visual changes
- **Calm Interactions**: Gentle hover effects and state changes

## Information Density
- **Clean Layouts**: Plenty of white space, uncluttered design
- **Clear Visual Hierarchy**: Obvious information organization
- **Large Touch Targets**: Minimum 44px touch targets for accessibility
- **Consistent Patterns**: Identical elements look and behave the same way
```

#### **Interaction Design Guidelines**
```markdown
# Interaction Design for Autism Community

## Feedback and Response
- **Immediate Feedback**: Clear, immediate response to all user actions
- **Success Indicators**: Obvious confirmation of completed actions
- **Error Prevention**: Preventing errors rather than just handling them
- **Gentle Corrections**: Non-punitive error messages with clear resolution

## Navigation and Flow
- **Consistent Navigation**: Same navigation patterns throughout app
- **Clear Location**: Users always know where they are in the app
- **Predictable Paths**: Common tasks follow expected workflows
- **Escape Routes**: Easy way to return to safe, known states

## Customization and Control
- **User Preferences**: Extensive customization options for individual needs
- **Sensory Controls**: Volume, animation, and visual intensity controls
- **Layout Options**: User control over information density and organization
- **Routine Customization**: Full control over daily routine structure and timing
```

### Routine Support Requirements

#### **Daily Reset and Consistency**
- **Reliable Daily Reset**: Automatic task reset at user-specified time
- **Preference Persistence**: User customizations remain stable across sessions
- **Routine Templates**: Easy creation and switching between different routine types
- **Backup and Restore**: Protection against losing established routines

#### **Progress and Motivation**
- **Clear Progress Tracking**: Visual progress indicators without pressure
- **Achievement Recognition**: Positive reinforcement for routine completion
- **Flexible Goals**: Accommodation for different energy levels and capabilities
- **Historical Tracking**: Optional review of routine adherence patterns

### Communication Standards

#### **Language Guidelines**
```markdown
# Communication Standards for Autism Community

## Language Principles
- **Direct and Literal**: Avoid idioms, metaphors, and implied meanings
- **Specific and Concrete**: Use precise language rather than vague terms
- **Consistent Terminology**: Same words for same concepts throughout app
- **Positive Framing**: Focus on what users can do rather than limitations

## Instruction Design
- **Step-by-Step**: Break complex tasks into clear, sequential steps
- **Visual Support**: Combine text instructions with visual examples
- **Clear Expectations**: Explicitly state what will happen when users take actions
- **Multiple Formats**: Text, visual, and interactive instruction options

## Error and Help Messages
- **Non-Judgmental**: Avoid language that implies user error or inadequacy
- **Specific Solutions**: Exact steps to resolve issues rather than general advice
- **Reassuring Tone**: Calm, supportive language that reduces anxiety
- **Clear Next Steps**: Obvious paths forward from any error state
```
